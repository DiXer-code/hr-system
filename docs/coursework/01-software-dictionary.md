# Словник програмного забезпечення

## Предметна область

- `Працівник` — співробітник компанії з особовою карткою, контактами та кадровим статусом.
- `Департамент` — структурний підрозділ компанії.
- `Посада` — штатна одиниця, яка належить до конкретного департаменту.
- `Кадрова історія` — журнал подій працівника: прийняття, переведення, звільнення.
- `Документ` — файл, прикріплений до особової картки працівника.
- `Відпустка` — запис про період законної відсутності працівника.
- `Табель` — облік відпрацьованих годин за конкретну дату.
- `Роль` — набір прав доступу в системі.
- `Користувач` — обліковий запис для входу до системи.

## Технічні терміни

- `CRUD` — набір базових операцій: створення, перегляд, редагування, видалення.
- `JPA` — Java Persistence API для роботи з реляційною БД через сутності.
- `Repository` — шар доступу до даних і SQL/JPQL-запитів.
- `Service` — шар бізнес-логіки, де зосереджені правила збереження й перевірки.
- `Controller` — веб-шар, який приймає HTTP-запити та формує модель для шаблонів.
- `Thymeleaf` — серверний шаблонізатор для HTML-інтерфейсу.
- `Transaction` — логічно завершена операція над БД, яка виконується атомарно.
- `ACID` — набір властивостей транзакцій: atomicity, consistency, isolation, durability.

## Стани працівника

- `Активний` — працівник виконує свої обов'язки.
- `У відпустці` — працівник тимчасово відсутній з причини відпустки.
- `Лікарняний` — працівник тимчасово відсутній через непрацездатність.
- `Звільнений` — картка працівника архівна, але історія зберігається.
