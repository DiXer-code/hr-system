# Функціональні вимоги

## Основні функції системи

1. Система має надавати форму входу для авторизованого доступу.
2. Система має дозволяти переглядати список працівників і виконувати пошук за ПІБ.
3. Система має дозволяти створювати, редагувати та видаляти картки працівників.
4. Система має зберігати кадровий статус працівника та дату звільнення.
5. Система має дозволяти прикріплювати до працівника документи і фотографію.
6. Система має дозволяти прив'язувати працівника до департаменту та посади.
7. Система має автоматично створювати записи в журналі кадрової історії.
8. Система має дозволяти вести довідник департаментів.
9. Система має блокувати видалення департаменту, якщо в ньому є працівники або позиції.
10. Система має блокувати видалення працівника, якщо по ньому існують відпустки або табелі.
11. Система має формувати аналітичну сторінку з агрегованими звітами.

## Нефункціональні вимоги

- Інтерфейс має бути адаптивним для desktop і mobile.
- Всі форми мають мати базову валідацію обов'язкових і форматних полів.
- Збереження працівника має виконуватися транзакційно.
- Структура проєкту має бути розділена на `controller`, `service`, `repository`, `entity`.
- Для тестового середовища система має запускатися без MySQL через H2 in-memory database.

## Обмеження

- Основна СУБД проєкту — MySQL.
- Цільова версія Java — `17`.
- Фронтенд побудований на серверному рендерингу, без окремого SPA-клієнта.
