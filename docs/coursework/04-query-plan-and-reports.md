# План запитів і звіти

## Реалізовані звіти в застосунку

На сторінці `/reports` реалізовані такі звіти:

1. Звіт по департаментах:
   кількість працівників у розрізі статусів і сумарний фонд зарплати.
2. Навантаження по позиціях:
   кількість активних призначень на кожну штатну одиницю.
3. Останні найми:
   вибірка останніх доданих працівників за датою прийняття.
4. Поточні відпустки:
   перелік активних записів із таблиці `Vacation`.
5. Останні кадрові події:
   журнал прийняття, переведення та звільнення з `JobHistory`.

## Індекси, що підтримують звіти

- `idx_employee_department` — використовується для групування працівників за департаментом.
- `idx_employee_position` — прискорює звіти по штатних одиницях.
- `idx_employee_status` — прискорює фільтрацію за статусами.
- `idx_job_history_employee_dates` — використовується в журналах і вибірках історії.
- `idx_vacation_period` — прискорює пошук активних відпусток.
- `idx_timesheet_employee_date` — прискорює суму годин за період.

## План виконання

Для ключових запитів слід використовувати `EXPLAIN`, наприклад:

```sql
EXPLAIN
SELECT d.name,
       COUNT(e.id) AS total_employees
FROM Department d
LEFT JOIN Employee e ON e.current_department_id = d.id
GROUP BY d.id, d.name;
```

Очікування від плану:

- `Department` читається послідовно або по первинному ключу;
- `Employee` при JOIN використовує індекс `idx_employee_department`;
- для запитів по історії та відпустках використовуються комбіновані індекси по датах.
